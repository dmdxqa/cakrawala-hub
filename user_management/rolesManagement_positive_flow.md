# Roles Management Work Flow

Defines the governance framework for creating, modifying, and configuring system roles. This workflow establishes Role-Based Access Control (RBAC) boundaries, ensuring that permissions across all modules—such as Work Orders, Sales Orders, and User Management—are strictly aligned with organizational and client hierarchies.

## Workflow

### Create New Role

1. Access to **User Access Management**
2. Open submenu **Roles Management**
3. Click **New Role** button
4. Fill in the field, here are the detail :

| Field | Input Field | Detail | Rules | Required |
| --- |  --- | --- | --- | --- |
| Role Name | Text | Name for the role | - | YES |
| Description | Free Text | Description for the role | - | NO |
| Status | Toggle | Switch toggle for Active or Inactive | - | YES |

5. Click **Save** button
6. Verify **New Role** is listed and able to select on other menu

### Edit Role

1. Navigate to **Action** section
2. Click open to select **Edit** button
3. Verify **Edit Role** is open
4. Update any field
5. Click **Save**
6. Verify the update is reflected

### Filter Function
Filter can be perform with **Status** and **Role Name**

### Status Toggle
Status can be updated through a **Toggle**
